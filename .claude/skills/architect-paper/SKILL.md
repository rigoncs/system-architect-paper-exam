# 系统架构设计师论文生成

根据软考高级系统架构设计师考试的论文要求，生成符合十段式模板的论文。

## 工作流程

按以下步骤依次执行，每完成一步向用户确认后再继续。

### 第一步：确定题目和主题

询问用户以下信息（如果用户已提供则跳过）：

1. **论文题目**：具体的题目或子题目是什么？例如"论云原生架构设计"、"论软件系统可靠性设计"等
2. **主题方向**：属于以下哪类？
   - 八大架构类（云原生、微服务、安全架构、面向服务架构、大数据架构、物联网架构、区块链架构、人工智能架构）
   - 软件工程类（敏捷开发、面向对象、测试运维、基于构件开发）
   - 系统质量属性类（可靠性、安全性、容错技术、性能优化）
   - 企业应用集成类（企业集成平台、应用集成模式）
3. **三个论点**：用户是否有指定的论点？没有则根据题目自动提炼三个分论点

### 第二步：确定项目参数

根据以下规范帮用户确定或编造合理的项目参数：

- **项目时间**：离当前时间 5 年以内，持续 6 个月以上，已完成
- **项目级别**：省市级、集团级，不写真实项目名称，写"某地级市""某医院""某公司""某研究所"
- **项目金额**：按团队人数 × 2万/月 × 持续月数计算，建议 500 万左右，不写整数（如 565 万、618 万）
- **团队规模**：建议 15-25 人
- **项目历时**：建议 8-18 个月

向用户展示参数方案，确认后再继续。

### 第三步：生成论文

按照十段式结构生成完整论文，严格遵循字数分配：

| 部分 | 段数 | 字数 | 说明 |
|------|------|------|------|
| 摘要 | 2段 | ~300字 | 第一段通用项目介绍，第二段回应子题目并介绍结构 |
| 背景 | 2段 | ~500字 | 第一段项目原因，第二段功能组成及过渡 |
| 正文 | 3段 | ~1200字 | 每段400字，回应三个子题目/论点 |
| 结尾 | 2段 | ~400字 | 第一段总结效果，第二段不足与展望 |

**总字数：2200-3000字**

生成后让用户审阅，支持针对性修改。

## 摘要模板

```
20XX年X月，我参与了某[单位类型]的[项目名称]项目的建设，并担任系统架构设计师，负责系统架构设计工作。该系统包括[三大功能模块]，能够[核心能力描述]。该项目总投入XXX万元人民币，历时XX个月，于20XX年XX月正式交付运行至今，受到了客户的一致好评。

本文结合笔者的实际工作经验，就该项目在[论文主题]方面的实践展开讨论。[简要概括三个分论点]，最后总结了项目中的不足之处及改进思路。
```

## 背景模板

**第一段（项目原因，~200字）：**
- 说明为什么要做这个项目（行业痛点、业务需求）
- 避免和论文主题直接相关，保持通用性

**第二段（功能组成 + 过渡，~300字）：**
```
我所在的公司成功中标该项目，并于20XX年XX月正式启动，我被任命为系统架构设计师，负责系统架构设计工作。该项目总投入XXX万元，建设周期从20XX年X月至20XX年X月，历时XX个月。系统采用[技术栈]。项目主要建设内容包括三大模块：一、[模块1]，完成[具体功能]；二、[模块2]，完成[具体功能]；三、[模块3]，完成[具体功能]。

[过渡段：点出项目难点或特点，引出论文主题和三个分论点]
```

## 正文结构要点

正文3段分别回应三个子题目/论点，每段约400字：

1. **第一段**：回应第一个子题目，阐述理论 + 项目实践
2. **第二段**：回应第二个子题目，阐述理论 + 项目实践
3. **第三段**：回应第三个子题目，阐述理论 + 项目实践

每段遵循"理论阐述 → 项目具体做法 → 取得效果"的结构。

## 结尾模板

**第一段（总结，~200字）：**
```
经过近XX个月的项目开发，[项目名称]顺利投入使用，[实际效果描述]，运行至今客户反馈良好。由于笔者及项目团队成员十分重视[论文主题相关的要点]，最终保证了该项目按质按量顺利交付。
```

**第二段（不足与展望，~200字）：**
```
当然，在本项目中，还有一些不足之处，比如：[列举1-2个小问题，切忌大问题]。不过，经过我后期的纠偏，并没有对项目产生什么影响。在后续的学习和工作中，我将不断充电学习，和同行进行交流，提升自己的专业技术水平，更好地完成系统架构设计的工作。
```

## 关键词库

为不同主题方向准备常用关键词：

- **云原生**：服务化、弹性、可观测、韧性、自动化、Kubernetes、CI/CD、链路追踪、容器编排
- **微服务**：服务拆分、独立部署、去中心化、容错、API网关、服务发现、配置中心
- **安全架构**：身份认证、访问控制、数据加密、安全审计、威胁建模、纵深防御
- **可靠性**：冗余设计、故障隔离、降级熔断、监控告警、灾备恢复
- **软件工程**：需求分析、架构设计、详细设计、编码实现、测试验证、部署运维

## 注意事项

1. 论文必须包含**三个论点**，分别对应正文三段
2. 以系统架构设计师的第一人称视角写作
3. 不要在论文中写"这是内部项目"
4. 项目背景尽量通用化，不要和论文主题绑定太紧
5. 选编的项目金额在500万左右，不写整数
6. 结尾的不足只能写小问题，不能写影响项目成败的大问题
7. 全文应有"我"作为系统架构设计师的视角贯穿始终
