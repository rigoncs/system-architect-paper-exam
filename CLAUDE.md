# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project overview

This is a preparation toolkit for the **系统架构设计师 (System Architect Designer)** exam paper section (软考高级). It contains a reusable skill for generating exam papers and a set of predicted paper templates for the May 2026 exam.

## Repository structure

```
papers/          # 5 predicted exam papers for May 2026 (ready to memorize)
.claude/skills/  # The architect-paper skill used to generate new papers
prompt.md        # Raw exam requirements and paper-writing rules (source of truth)
```

## The architect-paper skill

The skill at `.claude/skills/architect-paper/SKILL.md` generates complete exam papers following the **ten-section format (十段式)**. It is automatically triggered by keywords like 架构论文, 系统架构师, 软考论文 etc.

**Workflow:** The skill runs a 3-step process: (1) determine the topic and thesis points, (2) establish believable project parameters, (3) generate the full paper. It should confirm with the user at each step before proceeding.

**Key constraints the skill enforces:**
- Word count: abstract ~300, background ~500, body ~1200, conclusion ~400 (total 2200-3000)
- Project budget: ~500万, never round numbers (e.g., 565万 not 600万)
- Project timeline: 5 years from current date, duration 6+ months, already delivered
- Projects are anonymized: "某省级", "某医院", "某公司" — never real names
- Three thesis points (论点) are mandatory, each covered in one body paragraph
- Write in first-person as the 系统架构设计师 throughout
- Never describe it as an internal project
- Minor shortcomings only in the conclusion — never major project failures

**The reference paper** at `.claude/skills/architect-paper/references/cloud-native-example.md` shows the expected tone, paragraph structure, and level of technical detail.

## Working with papers

New papers are generated by invoking the skill (`/architect-paper`) or asking Claude to write a paper on a specific topic. Generated papers should be saved to `papers/`.

Existing papers in `papers/` are full 2200-3000 character essays covering Service Mesh, reliability design, Kappa architecture, AIOps, and distributed caching. They should be maintained in sync with the latest exam trends.
